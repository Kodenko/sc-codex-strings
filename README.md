# sc-codex-strings

Оверлей строк для **Star Citizen** — обогащает терминал контрактов и сканер руды
метками **состава чертежа**, **репутации** и **сигнатуры руды**, сгенерированными
из базы данных [sc-codex](https://sc-codex.ru).

> Сборка под билд `sc-alpha-4.9.0_live_12232306`. Проверяйте обновление после
> каждого патча игры — новые строки без свежего оверлея останутся оригинальными
> (ничего не сломается, просто без меток).

## Что делает

**Контракты** — к заголовкам в меню миссий дописываются подсказки:

```
Salvager Needed (Lrg. Special Order) <EM4>[Rep +150] [BP: Abrade Scraper Module, Cinch Scraper Module, ...]</EM4>
```

- `[Rep +N]` — сколько очков репутации даёт контракт.
- `[BP: …]` — **конкретный состав** пула чертежей-наград (реальные названия
  предметов, а не просто «есть пул»).

**Руда** — к названиям при сканировании добавляется сигнатура:

```
Quantainium (RS 3170, ×2)
Hephaestanite (RS 4180, ×6)
```

- `RS N` — базовая сигнатура сканера (scan value одного камня).
- `×N` — максимальный размер грозди (сколько камней в скоплении).

Охват: **774** строки контрактов + **26** руд.

## Чем отличается от аналогов (напр. StarStrings)

| Аналоги | sc-codex-strings |
|---|---|
| Метка `[BP]` по общей строке заголовка | Метка **по конкретному контракту** (per-UUID) из БД — меньше ложных срабатываний на общих заголовках |
| «есть пул чертежей» | **Реальный состав**: перечислены названия предметов |
| Ручная/полуручная разметка | **Генерация из БД** — воспроизводимо каждый патч одной командой |
| RS-сигнатуры руды вписаны вручную | RS **и размер грозди** из наших данных добычи |
| Публикуется весь `global.ini` (контент CIG) | Публикуется **только наш оверлей** (наши данные); базовый `global.ini` вы берёте из своей копии игры |

Такой подход чище по правам: мы не переупаковываем текст игры целиком, а
накладываем свои метки на вашу локальную копию.

## Установка (просто) — рекомендуется

1. Открой [**последний релиз**](../../releases/latest) и скачай `sc-codex-strings-….zip`.
2. Распакуй архив в **корень установки Star Citizen** — папку `StarCitizen/LIVE`.
   Внутри архива уже готовая структура:
   ```
   data/Localization/english/global.ini
   user.cfg
   ```
   Если `user.cfg` уже есть — не перезаписывай, а допиши в конец строку
   `g_language = english`.
3. Запусти игру.

Откат: удали `data/Localization/english/global.ini` и строку `g_language` из `user.cfg`.

> Проверяй новый релиз после каждого патча игры — метки завязаны на версию.

## Сборка вручную (для продвинутых)

Нужен Python 3 и распакованный из игры `global.ini`.

1. **Достаньте чистый `global.ini`** из своей копии игры (файл лежит внутри
   `Data.p4k`). Извлекается любым распаковщиком `.p4k`
   (например, [unp4k](https://github.com/dolkensp/unp4k)):
   путь внутри архива — `Data/Localization/english/global.ini`.

2. **Наложите оверлей:**
   ```bash
   python build/merge.py --base путь/к/global.ini --out global.ini
   ```
   Скрипт заменит значения только для наших ключей, остальное оставит как есть.

3. **Скопируйте в игру** (`LIVE` — корень установки Star Citizen):
   ```
   StarCitizen/LIVE/
   ├── user.cfg                                  (из этого репозитория)
   └── data/Localization/english/global.ini      (полученный на шаге 2)
   ```
   Если `user.cfg` уже есть — не перезаписывайте, а допишите в конец строку
   `g_language = english`.

4. Запустите игру.

## Пересборка оверлея

Оверлей `overlay/contracts.ini` генерируется из БД sc-codex. При выходе патча
перегенерируйте его и повторите шаг наложения.

## Правовая справка

- Возможность подмены локализации через `global.ini` разрешена CIG для
  community-переводов
  ([Community Localization Update, 2023-10-11](https://robertsspaceindustries.com/spectrum/community/SC/forum/1/thread/star-citizen-community-localization-update)).
- Названия предметов и заголовки контрактов — собственность Cloud Imperium
  Games. Этот репозиторий содержит только оверлей-метки, сгенерированные из
  открытых игровых данных; базовый `global.ini` вы предоставляете сами.
- Неофициальный фан-проект, не аффилирован с Cloud Imperium.
